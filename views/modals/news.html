Mountain Lion

Yes, we've heard it's been release. We are actually using it. And we've seen the incompatibility message that mail.app spits out regarding GPGMail.So, when is Mountain Lion support coming? The honest answer is: We don't know. Sandboxing requires a few changes to be made to GPGMail and before we come to that, there is another item above that on our priority list. And some more messages to make this longer.

GPGMail 2.0

We've had alphas of GPGMail 2 for a while now. The latest official release was GPGMail 2.0a30. We are today officially releasing a beta: GPGMail 2.0b5. It is a free download, but donations are recommended and very welcome. The more income we have via GPGTools the more time we can invest into this beloved project.

Funding

GPGTools is a spare time project. And only two of the four core-team members are currently working on it. The other two will return for sure but for the time being, GPGTools has a long to-do list and we need to prioritize in order to move things forward.

We are investigating options to ensure funding for GPGTools. An application for a funding is underway, but we have yet to hear back from that. Kickstarter is not an option, since all core-team members are from Europe and an US banking account is required. Right now donations are the easiest way to deal with that situation.

Workarounds

We know many of you rely on GPGTools and we do so ourselves. So we hate when things break just as much as you do. If you need to keeps things running here are two workaround suggestions:

  • 1) GPG Services

    We enabled GPGServices use in Mail.app. Meaning you can now encrypt and decrypt right in the mail window. Not beautiful, but works.

  • 2) Known Issues Thunderbird and Enigmail

    Install the latest Thunderbird and Enigmail add-on. Not beautiful, but works.
