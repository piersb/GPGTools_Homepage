  • Encrypt and sign messages with ease

    Simply click on the lock and sign buttons to encrypt and sign your messages. GPGMail will learn from past messages and automatically sets the encrypt and sign status of newmessages for you.

  • Automatically decrypts and verifies messages

    Encrypted messages are automatically decrypted, signed messages verified. Let GPGMail do the heavy lifting for you.

    GPG and S/MIME in love

    Switch between GPG and S/MIME in a second and after some learning time, let GPGMail do it for you.

  • Spot errors quickly

    If GPGMail fails to decrypt or verify a message you'll see a descriptive error message including hints at how to solve them.

    Be secure early on

    GPGMail will encrypt & sign your drafts, so they are never stored on your email server insecurely.

  • Choose the key to sign from

    Two of your keys share the same address? Select the one you want to sign from.

  • Create your own key

    First time users without an existing key, will be guided through the process of creating the first key pair.

  • Use the key servers

    Upload your public key to and search for your friends keys on the key servers.

    Web of Trust

    Sign keys to add to the web of trust. By signing keys you allow others to judge which keys are trustworthy.

  • Add more mail addresses

    If you use more than one mail address you can add UserIDs to an existing key, allowing to use more than one mail address per key.

    Import and Export keys

    Import via key servers, drag & drop or copy & paste. Export secret and public key or only the public part.

  • Revoke keys

    Create a revocation certificate for your keys. Revoke keys that are no longer in use.

        Let's Keep This Rolling Together!

        August 3rd, 2012

        Mountain Lion

        Yes, we've heard it's been release. We are actually using it. And we've seen the incompatibility message that mail.app spits out regarding GPGMail.So, when is Mountain Lion support coming? The honest answer is: We don't know. Sandboxing requires a few changes to be made to GPGMail and before we come to that, there is another item above that on our priority list. And some more messages to make this longer.

        GPGMail 2.0

        We've had alphas of GPGMail 2 for a while now. The latest official release was GPGMail 2.0a30. We are today officially releasing a beta: GPGMail 2.0b5. It is a free download, but donations are recommended and very welcome. The more income we have via GPGTools the more time we can invest into this beloved project.

        Known Issues
        • * missing setting to set default encryption / signing
        • * bug where the senders mail address ends up in bcc: and thus the sender receives the sent mail in his inbox

        Funding

        GPGTools is a spare time project. And only two of the four core-team members are currently working on it. The other two will return for sure but for the time being, GPGTools has a long to-do list and we need to prioritize in order to move things forward.

        We are investigating options to ensure funding for GPGTools. An application for a funding is underway, but we have yet to hear back from that. Kickstarter is not an option, since all core-team members are from Europe and an US banking account is required. Right now donations are the easiest way to deal with that situation.

        Workarounds

        We know many of you rely on GPGTools and we do so ourselves. So we hate when things break just as much as you do. If you need to keeps things running here are two workaround suggestions:

        • 1) GPG Services

          We enabled GPGServices use in Mail.app. Meaning you can now encrypt and decrypt right in the mail window. Not beautiful, but works.

        • 2) Known Issues Thunderbird and Enigmail

          Install the latest Thunderbird and Enigmail add-on. Not beautiful, but works.

        Summing things up:

        • Top Priority: GPGMail 2.0 final for 10.7
        • Next up: 10.8 compatibility
        Let's keep this rolling together!
        All the best and looking forward to some sweet releases,
        Steve, on behalf of the GPGTools team
